# Render

You can either render your PIXI React components in the DOM or through a custom createRoot call.

## In ReactDOM

This is the most common scenario


```jsx
import { Stage, Sprite } from '@inlet/react-pixi';
import { createRoot } from 'react-dom/client';

const App = () => (
  <div>

    {/* somewhere in your tree */}
    <Stage>
      <Sprite image="./bunny.png" x={100} y={100} />
    </Stage>

  </div>
);

const htmlRoot = document.getElementById('root');
const root = createRoot(htmlRoot);
root.render(<App />);
```

## Custom createRoot call

You can also render a React Pixi component tree directly using the `createRoot` function from `@inlet/react-pixi` and bypass ReactDOM entirely:


```jsx
import { createRoot, Text } from '@inlet/react-pixi';
import { Application } from 'pixi.js';

// Setup PIXI app
const app = new Application({
  width: 800,
  height: 600,
  backgroundColor: 0x10bb99,
  view: document.getElementById('container'),
});

// Use the createRoot to render a valid PIXI object into a PIXI container.
const root = createRoot(app.stage);
root.render(<Text text="Hello World" x={200} y={200} />);
```

Internally `react-pixi` keeps track of a `roots` list with containers, if you're removing/unmounting the container (or PIXI application),
it's advisable to tear it down correctly. Simply call `unmount` on the `root`:


```jsx
import { createRoot, Text } from '@inlet/react-pixi';
import { Application } from 'pixi.js';

const app = new Application({...});

const root = createRoot(app.stage);
root.render(<Text text="Hello World" />);

// clean up on unmount
// this removes the container from roots list
root.unmount();
```
